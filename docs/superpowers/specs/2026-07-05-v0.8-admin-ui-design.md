# V2 Admin UI 设计（v0.8.0）

**日期：** 2026-07-05  
**状态：** 已批准  
**版本目标：** v0.8.0 - Admin UI

---

## 1. 背景

V2 已完成 v0.1–v0.7：Chat / RAG / Prompt / OCR / Workflow / Agent Router / Streaming + Tool Calling。

当前所有操作依赖 API 调用或 Swagger (Knife4j)，无可视化管理界面。下一项为 **v0.8.0 Admin UI** — 独立 React 前端工程，构建时嵌入 jar，提供 5 个核心模块的可视化管理。

**已确认决策（brainstorming）：**

| 项 | 选择 |
|----|------|
| 技术栈 | React 18 + TypeScript + Vite |
| UI 范围 | 5 模块：Chat / RAG / Prompt / Tools / Login |
| 部署方式 | 构建时嵌入 jar（保留「单 jar 启动」卖点） |
| 设计美学 | 暗色科技风（Dark Tech） |
| 布局 | 顶部导航 + 全屏工作区 |
| 状态管理 | React Context + hooks（不引入 Redux） |
| SSE 库 | `@microsoft/fetch-event-source` |
| 字体 | Space Grotesk（标题）+ JetBrains Mono（正文，等宽科技感） |
| 强调色 | 霓虹三色：青 (#00d4ff) / 品红 (#ff2d92) / 绿 (#00ff88) |
| 后端 UI | 不做 Workflow / Router UI（已有 Swagger） |
| 多语言 | 仅中文 |
| 主题 | 仅暗色 |

---

## 2. 目标

- 新增 `frontend/` 独立 React 工程，5 个模块可视化
- Maven 构建时 `npm build` + 复制 `dist/` 到 `static/`，最终单 jar 包含前后端
- Chat / RAG 对接 v0.7 SSE 流式端点，Tool 调用可视化
- 暗色科技风视觉语言，远离「通用后台」平庸感
- 不破坏现有 API 与 `verify.ps1` 已有验收项

---

## 3. 非目标（v0.8 不做）

- Workflow / Agent Router 的 UI（已有 Swagger）
- 多语言（仅中文 UI）
- 主题切换（仅暗色）
- 移动端深度适配（仅基础响应式）
- Token 自动刷新
- 用户管理 UI（仅 admin 单账号）
- 多租户

## 5. 视觉语言（暗色科技风）

### 5.1 色彩系统

```css
--bg-base:        #0a0e14    /* 主背景 */
--bg-panel:       #11161f    /* 卡片/面板 */
--bg-elevated:    #1a2030    /* 悬浮元素、输入框 */
--border:         #1f2a3d    /* 边框 */
--border-glow:    #2a3a55    /* hover 发光边框 */

--text-primary:   #e6edf3    /* 主文字 */
--text-secondary: #8b96a8    /* 次要文字 */
--text-muted:     #4a5568    /* 占位符 */

--accent-cyan:    #00d4ff    /* 主强调 — Chat / 按钮 / 链接 */
--accent-magenta: #ff2d92    /* 次强调 — Tool 调用 / 错误 */
--accent-green:   #00ff88    /* 成功 / RAG / Tool 结果 */
--accent-amber:   #ffaa00    /* Prompt 模块 / 警告 */
```

**规则：** 90% 深色 + 8% 文字 + 2% 霓虹强调。每个模块一个主导强调色：Chat=青、RAG=绿、Prompt=琥珀、Tools=品红。

### 5.2 字体

```css
--font-display: "Space Grotesk", sans-serif;     /* Logo / 大标题 */
--font-body:    "JetBrains Mono", monospace;      /* 正文（等宽科技感） */
```

JetBrains Mono 作为正文是关键差异化：等宽字体强化「终端/IDE」气质，让 Chat 流式输出有「代码执行」的科技感。

### 5.3 关键视觉元素

| 元素 | 规格 |
|------|------|
| 顶部导航条 | 56px 高，深色 + 底部 1px 霓虹渐变线；活跃模块有底部霓虹下划线 + 微弱发光 |
| 卡片/面板 | `bg-panel` + 1px `border` + 8px 圆角；hover 时 `border-glow` + 微弱外发光 |
| Chat AI 消息 | 左对齐 + 左侧 2px 青色竖线 + 等宽字体，无气泡 |
| Chat 用户消息 | 右对齐 + `bg-elevated` 气泡 |
| 流式光标 | 青色闪烁 `▊`，停在下一段待生成位置 |
| Tool 调用卡片 | 品红边框 + 半透明品红背景；折叠态一行；展开显示参数+结果+耗时；调用中边框流光 |
| 按钮 Primary | `bg-cyan` + 黑字 + hover 发光；4-6px 圆角保持工程感 |
| 按钮 Secondary | 透明 + `border` + hover `border-glow` |

### 5.4 动效

| 场景 | 动效 |
|------|------|
| 页面加载 | 模块内容 staggered fade-in（每个 +50ms delay） |
| Chat 流式 | 文字逐字显现 + 光标闪烁 |
| Tool 调用 | 卡片从上方滑入 + 边框流光 |
| 路由切换 | 旧内容 fade-out → 新内容 fade-in（150ms） |
| hover | 边框发光过渡 + 微弱位移 |

### 5.5 SSE 事件渲染规则

| 事件 | 渲染 |
|------|------|
| `start` | 隐藏（内部记录 conversationId） |
| `chunk` | 追加到当前 AI 消息 |
| `tool_call` | 渲染 ToolCallCard（调用中态） |
| `tool_result` | 更新 ToolCallCard 为完成态 |
| `done` | 移除流式光标；RAG 时渲染 `metadata.sources` |
| `error` | 顶部品红 toast + 错误内容 |

## 6. 模块设计

### 6.1 登录页 `/login`

- 全屏深色 + 中心卡片 + 背景微弱网格纹理
- Logo 用 Space Grotesk 大号字体
- Username / Password 输入框 + 「→ Enter Console」青色发光按钮
- 默认提示 `admin / admin123`
- 登录成功 → JWT 存 `localStorage` → 跳转 `/chat`
- 错误：输入框品红边框 + 错误文字

### 6.2 Chat `/chat`（核心模块）

- 顶部导航 + 全屏 Chat 工作区
- 消息列表：AI（左+青色竖线，无气泡，等宽）/ 用户（右对齐，气泡）
- 流式光标青色闪烁
- Tool 调用：品红卡片，折叠/展开，显示参数+结果+耗时
- 底部输入框 + `Enable Tools` 开关（对应 `enableTools`）

**核心组件：**
- `SseEventStream`：封装 fetch-event-source，解析 6 类事件
- `ChatMessage`：渲染 AI vs 用户
- `ToolCallCard`：品红折叠卡片
- `StreamingCursor`：青色闪烁 `▊`

### 6.3 RAG 知识库 `/rag`

左右分栏：
- **左侧（30%）：文档管理**
  - Upload 按钮（拖拽 / 点击）
  - 文档列表：文件名 + 大小 + 删除按钮
  - 空状态：虚线边框 + 「Upload first document」
- **右侧（70%）：RAG 流式问答**
  - 复用 Chat 流式组件
  - `done.metadata.sources` 渲染为绿色引用来源卡片

### 6.4 Prompt 管理 `/prompts`

左右分栏：
- **左侧（30%）：Prompt 树**
  - 按 key 分组（chat / database.agent / rag.chat）
  - 每项显示 type + 激活版本（●）
- **右侧（70%）：版本管理 + 编辑器**
  - 版本列表（active 高亮 + Activate 按钮）
  - 等宽字体编辑器（琥珀色边框聚焦）
  - `Save as new version` + `Render Preview`（调用 `/api/prompts/render`）

### 6.5 Tool 概览 `/tools`

- 纯展示页，对接 `GET /api/tools`
- 每张卡片边框颜色对应模块强调色（queryDatabase=品红、searchKnowledge=绿、calculateArea=青）
- `[READ]` 标签表示只读 Tool
- 参数 schema 折叠展示

## 7. API 层与状态管理

### 7.1 API 封装 `lib/api.ts`

- `ApiClient` 类，统一 `request<T>()` 方法
- JWT 从 `localStorage` 读取，注入 `Authorization: Bearer`
- 401 → 清除 token + 重定向 `/login`
- 模块化方法：`auth.login`、`rag.*`、`prompts.*`、`tools.list`

### 7.2 SSE 封装 `lib/sse.ts`

- `streamChat(body, onEvent, onError)` — POST `/api/chat/stream`
- `streamRagChat(body, onEvent, onError)` — POST `/api/rag/chat/stream`
- 使用 `@microsoft/fetch-event-source`（支持 POST + headers）
- `onmessage` 解析 JSON → 调用 `onEvent`
- `onerror` 不重连（throw 停止）

### 7.3 状态管理

```
src/context/
├── AuthContext.tsx       登录状态 + JWT
└── ToastContext.tsx      全局错误/成功提示
```

- **AuthContext：** `isAuthenticated` / `username` / `login()` / `logout()`，包裹 App，未登录路由重定向 `/login`
- **ToastContext：** 顶部居中 toast，3 秒自动消失，错误=品红/成功=绿/信息=青
- **页面状态：** 各页面 `useState` 管理自己数据，不跨页面共享

### 7.4 开发时 Vite Proxy

```typescript
// vite.config.ts
server: { proxy: { '/api': 'http://localhost:8080' } }
```

开发：`npm run dev`（5173）+ 后端 jar（8080），`/api/*` 自动转发。

## 8. 构建集成

### 8.1 Maven `frontend-maven-plugin`

根 `pom.xml`：
- `install-node-and-npm`（Node v20）
- `npm install`
- `npm run build`

### 8.2 复制产物

`starter-web/pom.xml` 加 `maven-resources-plugin`：
- `process-resources` 阶段复制 `frontend/dist/*` → `src/main/resources/static/`

### 8.3 SPA 路由 fallback

`starter-web` 新增 `WebConfig.java`：
- `/chat`、`/rag`、`/prompts`、`/tools`、`/login` → `forward:/index.html`
- `/api/**` 由 Controller 处理，不 forward

### 8.4 SecurityConfig 调整

放行前端路由 + 静态资源：
```java
.requestMatchers("/", "/index.html", "/chat", "/rag", "/prompts", "/tools", "/login").permitAll()
.requestMatchers("/assets/**", "/static/**").permitAll()
```

### 8.5 构建命令

```bash
mvn clean package -DskipTests
# → 自动 npm install + npm build + 复制 dist/
java -jar starter-demo/target/starter-demo-0.1.0-SNAPSHOT.jar
# → http://localhost:8080 直接访问 Admin UI
```
